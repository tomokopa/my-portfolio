ポートフォリオサイト（当サイト）

サービス概要

当サイトでは私（トモコ）の作成したWebサービスを御確認いただけます。

サービス作成理由

  • 私に興味を持ってくださった方にスキルや実績を知っていただくため。
  • 当サイトの作成を通して、Webサービス関連の仕組み（サーバーへのデプロイやフレームワークなど）について学びたかったため。

苦労した点・こだわりの点

レスポンシブルデザイン

レスポンシブルデザインを意識し、素早く導入可能なフレームワーク「Bootstrap」を活用しております。
パソコンとスマートフォンの両方で確認しながら作成しました。
Bootstrapの機能を利用した主な箇所は、

  • ナビゲーションバー（ヘッダー）
  • テーブル（Profile）
  • カード（Skill）
  • フォーム（Contact）

です。

コンタクトフォームの設置

ポートフォリオ内にコンタクトフォームを設置しました。
お問い合わせ内容はgmailで受け取ります。

設置を通して、サーバーからデータを受け取る方法を学びました。

.envファイルからの環境変数の読み込み

python-dotenvモジュールをインストールすることで、ローカル環境と本番環境でそれぞれの.envファイルから環境変数を読み込むようにしました。
これにより、環境変数をコンソールに打ち込む手間が減らすことが出来ました。

本番環境であるPythonanywhereへの.envファイルの適用方法がなかなか分からず、苦労しました。
Pythonanywhereの公式サイトのヘルプページがとても参考になりました。

The PythonAnywhere help pages

この作業を進めるうちに、Git、Github、.gitignoreの理解を進めることが出来ました。

使用技術と選定理由

Python 3.9

Pythonは初めて本格的に学習した言語であり、知識があるため。
Pythonを学習した理由は、

  • 初心者でも学びやすい、という情報をインターネットから得たこと
  • 機械学習に興味があったこと

です。
バージョンは、Pythonanywhereが対応しているバージョンの中で開発時に最新だった3.9を選定しました。

Django 2.2.4

PythonのWebフレームワークの中で、Djangoはチュートリアルや解説サイトが多かったため。
バージョンは、参照したDjangoのチュートリアルで使用されていた2.2.4を選定しました。

Bootstrap5

参照したDjangoのチュートリアルで紹介されていたため。
また、実際に試したところレスポンシブルデザインが簡単に素早く導入可能で、公式の解説用ドキュメントも充実していました。
バージョンは開発時に最新だった5を選定しました。

Pythonanywhere

参照したDjangoのチュートリアルで紹介されており、無料で利用可能なサーバーだったため。
SSLの設定が簡単なことも魅力だと思いました。

GitHub

参照したDjangoのチュートリアルで紹介されていたため。
GitHubからPythonanywhereにコードをプルすることでサイトを公開しています。
GitHubでWebサービスを公開し、そのコードも公開出来ることが便利だと思いました。
